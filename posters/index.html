High-Performance Computing Workflow for Rapid-Response Physics-Based 3D Earthquake Simulations Integrated In Data Lake

Dynamic rupture simulations are the gold standard for earthquake modeling, but setting them up after an event can take weeks. This work presents a fully automated rapid-response workflow that delivers a first simulation result within an hour of an earthquake occurring. The approach relies on a pre-computed catalog of physics-based 3D rupture scenarios for the Alto Tiberina fault (Italy), generated using SeisSol on the SuperMUC-NG supercomputer. When an earthquake occurs, the workflow automatically retrieves observational data from the TABOO Near-Fault Observatory, compares it to the catalog, and outputs the closest-matching scenario within minutes. For FAIR data access, the catalog is stored in the Geo-INQUIRE Simulation Data Lake (SDL), enabling long-term, HPC-close storage with API access for programmatic workflows. The workflow is implemented in Common Workflow Language and is available for adoption in other regions and with other simulation packages.
